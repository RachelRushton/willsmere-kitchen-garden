# Willsmere Kitchen Garden

A small static website for the Willsmere Kitchen Garden community group.

- **Frontend:** React 18 + Vite + React Router
- **Backend:** None required — the Facebook feed is embedded client-side using
  Meta's official [Page Plugin](https://developers.facebook.com/docs/plugins/page-plugin/),
  so the whole site can be hosted as static files.
- **Hosting:** GitHub Pages, deployed automatically by GitHub Actions.

## Pages

- **Home (`/`)** — welcome message and the live Facebook feed.
- **About (`/about`)** — info about the group, what they do and how to get involved.

## Local development

```powershell
npm install
npm run dev
```

Then open the URL Vite prints (usually <http://localhost:5173/willsmere-kitchen-garden/>).

To build a production bundle locally:

```powershell
npm run build
npm run preview
```

## Configuration

### Facebook page URL

Edit the `pageUrl` default in `src/components/FacebookFeed.jsx` (or pass it as a
prop from `src/pages/Home.jsx`) to the full URL of the group's Facebook page,
e.g. `https://www.facebook.com/WillsmereKitchenGarden`.

> The Facebook Page Plugin only works for **public Pages**, not personal
> profiles or private groups. If the community uses a private group, the feed
> can't be embedded — link out to it instead.

### Repository / GitHub Pages base path

`vite.config.js` sets `base: '/willsmere-kitchen-garden/'` so asset URLs work
when served from `https://<user>.github.io/willsmere-kitchen-garden/`.

If you rename the repo, change that value to match. If you set up a custom
domain or use a user/organisation site (`<user>.github.io`), change it to `'/'`.

The site uses `HashRouter`, so deep links like `/#/about` work on GitHub Pages
without any server-side rewrites. A `404.html` copy of `index.html` is also
generated by the workflow as an extra safety net.

## Deploying

1. Push this repository to GitHub (`main` branch).
2. In the repo on GitHub, go to **Settings → Pages** and set
   **Build and deployment → Source** to **GitHub Actions**.
3. Every push to `main` will trigger `.github/workflows/deploy.yml`, which
   builds the site with Vite and publishes `dist/` to GitHub Pages.

The deployed URL will appear in the workflow run summary, and on the
Settings → Pages screen.

## Why no C# backend?

The two requirements — a Facebook feed and a static info page — can both be
served as static files. The Facebook feed is loaded directly in the user's
browser by Meta's SDK, so there's nothing for a backend to do. This keeps
hosting free on GitHub Pages and removes a whole class of things that could
break. If later on you need things like a contact form, members area, or a
custom events API, an ASP.NET Core Web API can be added and called from the
React app via `fetch`.
